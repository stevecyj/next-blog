# 3-7 元件狀態（Component State）

## 核心內容

React 元件的畫面更新由**狀態（state）**驅動，而非普通 JavaScript 變數。

- 普通 `let` 變數在事件處理器中被修改後，React 完全不知道，不會觸發重新渲染。
- 只有透過 `useState` Hook 管理的值，React 才會追蹤其變化並在改變時重新渲染元件。
- `useState` 回傳一個陣列：`[狀態值, 更新函式]`，使用陣列解構取得兩者。
- 狀態值只能透過更新函式（如 `setLabel`）修改，直接賦值無效。
- 呼叫更新函式後，React 重新執行元件函式，依最新狀態更新畫面；重新渲染之間狀態不會重置。

**命名慣例**：狀態變數用描述名稱（`label`），更新函式以 `set` 開頭加 CamelCase（`setLabel`）。

**Hook 規則**：所有 Hook 必須在元件函式的最頂層呼叫，不可放在迴圈、條件判斷或巢狀函式內。

## 開發上可採取的行動步驟

1. 在元件頂部以 `import { useState } from 'react'` 引入 Hook。
2. 宣告狀態：`const [label, setLabel] = useState('show')`，初始值傳入 `useState`。
3. 在事件處理器中呼叫 `setLabel(newValue)` 更新狀態，不要直接賦值給 `label`。
4. 若需切換值，在更新函式內依目前狀態決定新值：
   ```tsx
   setLabel(label === 'show' ? 'hide' : 'show')
   ```
5. 確認所有 `useState` 呼叫都在元件函式的最頂層，不在任何條件或迴圈內。

## 我可以立刻採取的實作清單

- [ ] 在現有元件中找一個用普通變數控制 UI 的地方，改用 `useState` 重寫。
- [ ] 新增一個按鈕，點擊後切換顯示/隱藏文字，驗證狀態在重新渲染後不被重置。
- [ ] 故意在條件判斷內呼叫 `useState`，觀察 React 拋出的 Hook 規則錯誤訊息。

## 總結

React 透過 `useState` Hook 追蹤元件狀態：狀態改變 → 元件重新渲染 → UI 更新。普通變數的修改對 React 不可見，無法觸發渲染。所有 Hook 必須在元件頂層無條件呼叫。
